主线流程图 · 业务场景独立说明
01 客户入口 WEB 注册、电话接待、到店下单，统一进入销售链路。
→
02 订单处理 WEBADMIN / OMS 审核订单、确认价格、库存与特殊要求。
→
03 仓储执行 WMS 拣货、复核、出库、自取或配送，完成履约动作。
→
04 库存同步 出入库后回写库存，保证 WEB 展示与仓储数据一致。
→
05 财务闭环 QBO 自动开票、冲销退款和成本同步，形成财务闭环。
1
客户入口
WEB / 电话 / 到店
新客注册开户WEB 提交注册，WEBADMIN 审核资质并激活权限。
老客线上下单客户登录 WEB，选品、数量、PO 后提交订单。
电话 / 到店接待客服确认客户、需求、价格和库存。
2
订单处理
CORE-OMS / WEBADMIN
订单审核校验折扣、VIP、轮胎税、客户备注和特殊要求。
电话代客做单OMS 中代客下单，报价与库存确认后推进出库。
特殊订单处理取消、退换货、调货在 OMS 中形成明确单据。
3
仓储执行
CORE-WMS
拣货下架打印拣货单，扫码确认商品与数量。
配送 / 自取按交付方式记录出库，完成履约动作。
入库 / 调拨采购、退货、调货回仓后核对、上架、记录货位。
4
库存同步
CORE-CSM → WEB
出库扣减库存销售出库后实时扣减库存并同步前端。
入库增加库存收货、退货归还、上架后库存回写。
调货库存协调跨仓或跨地区调拨保持库存状态一致。
5
财务闭环
QBO
产品及价格同步商品、规格、状态和不同客户价格同步到 QBO。
库存同步出入库后同步库存变化，保持前后端一致。
发票生成订单完成后在 QBO 生成发票并回写状态。
数据流向
订单流

客户需求进入 WEB 或 OMS，审核后转换为仓储可执行的拣货、出库或特殊处理单据。

库存流

WMS 负责出入库动作，CSM 汇总库存变化并同步回 WEB，保证前端库存展示一致。

财务流

QBO 根据出库、退货、取消和入库成本更新财务记录，形成销售与成本闭环。

基础数据
核心主数据
轮胎、轮毂、客户、订单、适配、库存
轮胎基础信息 规格、品牌、季节、花纹、价格和展示信息。
轮毂基础信息 尺寸、孔距、中心孔、偏距、品牌和车型适配。
客户基础信息 客户主体、等级、账期、联系人和地址信息。
订单基础信息 订单编号、客户、状态、金额、仓库和交付方式。
车型适配信息 年份、品牌、车型、配置、原厂尺寸和适配规则。
库存基础信息 仓库、库位、数量、批次、可售库存和锁定库存。
配套基础数据
轮胎周边配件与门店包装类
Weights / 平衡块 卡扣式、粘贴式两类平衡块，支持常规与 AW。
Bolts / 螺栓 按尺寸、长度、驱动位和座型统一维护。
Nuts / 螺母 带锁与不带锁螺母，按尺寸和包装清单管理。
Spacer / 垫片 螺栓式与套装式垫片，按厚度和孔距区分。
Valve / Stem 标准气嘴与带传感器版本，按长度管理。
Center Ring / 中心环 按材质、外径、内径和颜色快速识别。
Wiper Blade / 雨刮片 按型号和长度维护，便于门店快速选配。
Tire Bag / 轮胎袋 按尺寸和 logo 版本管理，便于出库包装。
特殊业务场景
取消发票产生前才能操作取消；若已产生发票，则需走财务冲销流程。
修改发票产生前才能操作修改；若已产生发票，则需重新生成发票。
退货需要手工创建 Credit Memo，并拿单据完成退货；WMS 处理回仓后再同步财务。
零售 WALK-IN个人开通账号后，按照流程下单、拣货、出库并生成发票。
紧急加单按照系统流程创建相关订单信息，并线下与配送团队沟通。
调货通过调货单功能，按照需求进行调货操作。
