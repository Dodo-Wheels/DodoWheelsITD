DodoWheels 运营管理总规范

版本：V1.1（增强版） | 生效日期：2025年12月

一、 供应链前端流程规范：采购、入库与上架

1.1 采购与产品信息建立

编号 流程步骤 责任人/部门 操作细则
1.1.1 采购订单下达 (PO) 采购（如：@Pan） 根据库存需求和销售预测下达PO。
1.1.2 产品信息建立 信息管理（TIM 或 部门主管） 采购订单下达后，需及时将**产品信息（型号、SKU）**告知TIM，安排**建立或更新产品信息**。

1.2 入库、验收与上架规范

编号 流程环节 执行细则 责任人/单据
1.2.1 到货签收与验收 货物到达，仓库人员需核对数量、型号与PO单/通知单是否一致。确认无误后方可签收。 仓库人员 / 签收单
1.2.2 系统入库登记 **DodoWheels主仓：** 货物入库至**备货区**。
**Publico仓库：** 货物入库至**T0位置**。系统同步登记库存数量。		 仓库人员 / WMS系统
1.2.3 货物上架 仓库人员应根据库存管理原则，将入库货物移动到指定的排位（A-P排）。**新入库无销售的货物，应优先上架归位。** 区域负责人（涂/孙/丁/许师傅）
1.2.4 财务结算单据 将**已签收的单据**及时移交财务部门（@zhiyou, @Pan, @Susan），用于**采购发票录入**。 仓库人员 / 财务部门

二、 履约后端流程规范：拣货、出库与物流

2.1 销售订单处理与拣货

编号 流程环节 执行细则 责任人/单据
2.1.1 订单下达 @小林 在群内下单给客户，操作人员创建**dodo---publico**内部销售单。 销售/操作人员
2.1.2 打印调拨/销售单 仓库（调出仓）收到ESTIMATE调货单或内部销售单后，**必须打印**并作为拣货指令。 仓库人员（@Mention All）
2.1.3 拣货与备货 区域负责人（涂/孙/丁/许师傅）根据指令，从对应的排位中准确拣取货物。针对调货单，必须先**安排备货**。 区域负责人
2.1.4 防抢占机制 **重要：** 收到调货单/销售单后，立即将货物移至备货区，避免因个别尺寸数量小，被后续其余订单抢占。

2.2 出库与物流规范

编号 流程环节 执行细则 责任人/单据
2.2.1 出库复核 货物出库前，仓库人员必须与**销售单**（面向客户/司机）或**ESTIMATE调货单**进行**二次复核**，确保型号和数量准确。 仓库人员
2.2.2 单据用途分离 **销售单**仅用于司机与客户提货，**严禁**拿销售单作为仓库间调货凭证。仓库调货一律使用**ESTIMATE单**。 所有人员
2.2.3 物流交接 货物装车时，仓库人员需核对司机或物流承运方提供的凭证，确保货物准确交付给正确的承运方。 仓库人员 / 物流承运方
2.2.4 系统出库确认 货物实际离开仓库后，必须及时在系统内进行**出库确认**，更新库存数据。 仓库人员 / WMS系统
